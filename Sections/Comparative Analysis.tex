\section{Comparative Analysis}
\label{chap:comparative}

This chapter synthesises the findings from the individual tool evaluations in
Chapter~\ref{chap:practical-evaluation} into a structured comparison. The
comparison is organised around three questions drawn from the Fase~3 task
specification: how the tools compare against the evaluation criteria
(Section~\ref{sec:eval-criteria-comparison}), how well they align with
established framework requirements (Section~\ref{sec:framework-alignment}),
and which tools are best suited to which organisational contexts
(Section~\ref{sec:org-suitability}).

\subsection{Comparison Against Evaluation Criteria}
\label{sec:eval-criteria-comparison}

This section compares the evaluated tools against the criteria set defined in
Section SECTION-REF, % TODO: adjust label to match 
which in turn is derived the frameworks covered in the literature review (ISO/IEC~27005,
NIST~SP~800-37, OCTAVE Allegro, FAIR, and ENISA guidance). The criteria are
grouped into five categories: methodology support, functionality,
integrations and reporting, user experience, and administrative factors.
Each category is presented as a separate table to keep the comparison
readable and to allow targeted discussion of findings.

Cell values use the following convention: \textbf{Yes} indicates full
support, \textbf{Partial} indicates the capability is present but limited or
requires manual configuration, \textbf{No} indicates the capability is
absent, and \textbf{N/A} indicates the criterion does not apply to the
tool's scope. Where a simple yes/no answer would be misleading, a short
descriptive phrase is used instead.
